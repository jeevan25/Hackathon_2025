Pytest-based API testing framework uses a specific convention for generating two files per endpoint: 
1. A helper file under `apiLibrary/open_api/`.
2. A test file under `tests/backend_tc/open_api/`.

Test files include full test classes using pytest, allure, and internal utilities like `GenerateCredentials`, `CSAPAPI`, and helper classes (e.g., `OpenAPIDocLibrary`). Data for tests is sourced from `dataDirectory/common_data.py`, and API status codes are defined in `dataDirectory/api_status_codes.py`.
Helper files encapsulate API-specific payload construction logic. The user will provide endpoint definitions and schemas for which they want automated test generation.
The following is the an example test file, you will notice general conventions

from utilities.utils import Util
class OpenAPIDocLibrary:

    def create_folder_payload(self, folder_name, description="", group_id = "", group_edit_access =  False , individual_email="", individual_edit_access = False, parent_folder_id = ""):
        payload = {
        "folder_name": folder_name,     #payload should be made based on the endpoint schema that varies for each endpoint
        "description": description,   #payload should be generated without hardcoding any values, and keeping as many fields dynamic as possible
        "parent_folder_id": parent_folder_id
        }
        if group_id != "":
            payload["recipient_groups"] = [{
            "group_id": group_id,
            "edit_access": group_edit_access   
        }]
        if individual_email != "":
            payload["individual_recipients"] = [{
            "email" : individual_email,
            "edit_access": individual_edit_access}]
        return payload

Endpoint: /csap/v1/allowed_indicators/
Method: POST
Authentication: Required

Payload Schema:
{
  "threat_indicators": "string"   # Required, comma-separated list of indicators (e.g., IPs, domains, hashes)
}

Key Notes:
- "threat_indicators" is a mandatory field.
- Accepts multiple indicators in a comma-separated string.
- The system categorizes each indicator (IP, domain, hash, etc.) automatically.

allowed_types = ["domain","ip","url","email","sha1","sha256","md5","cve","ipv4_cidr"
]

Expected Response:
{
  "sha256": [],                    # List of SHA-256 indicators added to the allow list
  "domain": [ "exampledomain.com" ],  # List of domain indicators added
  "sha1": [],                      # List of SHA-1 indicators added
  "url": [],                       # List of URL indicators added
  "ip": [ "1.1.1.1" ],             # List of IP address indicators added
  "cve": [],                       # List of CVEs (Common Vulnerabilities and Exposures) added
  "email": [],                     # List of email indicators added
  "md5": [],                       # List of MD5 hash indicators added
  "ipv4_cidr": [],                 # List of IPv4 CIDR indicators added
  "blocked_indicators": [],       # Indicators that were previously allowed but are now inactive
  "duplicate_indicators": []      # Indicators already existing in the allow list
}

Error Scenarios:
- Missing "threat_indicators" field returns 400 Bad Request.
- Adding same indicator again should return the same indicator in 'duplicate_indicators' in the response
